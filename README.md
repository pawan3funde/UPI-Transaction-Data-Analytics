# UPI Transaction Analytics & Fraud Monitoring System

## Project Overview

This project analyzes over **100,000 UPI transactions** to understand customer behavior, transaction performance, fraud patterns, merchant risk, and customer satisfaction. Using **Python, SQL, and Power BI**, the project transforms raw transaction data into actionable business insights through statistical analysis and interactive dashboards.

---

## Objectives

* Analyze transaction volume and trends
* Monitor fraud and risk patterns
* Evaluate customer satisfaction
* Assess merchant performance
* Build KPI-driven dashboards for decision-making

---

## Dataset Summary

The project integrates seven datasets:

* Customer Master
* UPI Transaction History
* UPI Account Details
* Device Information
* Merchant Information
* Fraud Alert History
* Customer Feedback Surveys

### Data Scale

* 100,000 Transactions
* 10,000 Customers
* 500 Merchants
* 2,000 Fraud Alerts
* 4,000 Feedback Records

## Key Insights

* Send and Receive transactions contribute nearly 70% of total activity.
* Fraud rate is approximately 2%.
* QR Code transactions show higher fraud exposure.
* Transaction-related issues are the most common customer complaints.
* Most customers reported satisfaction scores of 4 and 5.
* Transaction activity is evenly distributed across regions and device types.

---

## Business Recommendations

* Strengthen QR transaction security.
* Improve transaction failure monitoring.
* Enhance fraud detection for high-risk customers.
* Improve customer support for transaction-related issues.
